# Project Structure

Every file and directory orai creates, and what each one does.

---

## Overview

After running `orai init myapp` and `orai plan myapp`, your project looks like this:

```
myapp/
  CLAUDE.md
  app/
    .agents/
      project.json
      agents.json
      Skills.md
      CONTEXT.md
      tasks/
        phase1.json
        phase2.json
        phase3.json
      scripts/
    src/
      ...
    package.json  (or pyproject.toml)
```

---

## Files explained

### `CLAUDE.md`

Located at the project root. Contains coding guidelines that Claude reads automatically when working inside the project.

This file tells the AI:
- Architecture rules for the project
- Coding style expectations
- Available commands (test, lint, build)
- Where to find task definitions

You can edit this file to add your own rules. Claude respects whatever you write here.

### `app/`

The working directory where all generated code lives. When orai runs a task, it sets the working directory to `app/`, so all file paths in task prompts are relative to this folder.

### `app/.agents/project.json`

Project metadata. Created during `orai init`, updated during `orai plan`.

```json
{
  "name": "myapp",
  "project_type": "nextjs",
  "created_at": "2025-01-15T10:30:00Z",
  "phases": [1, 2, 3],
  "current_phase": 1
}
```

| Field | Description |
|-------|-------------|
| `name` | Project name |
| `project_type` | Template used (`nextjs` or `python`) |
| `created_at` | When the project was created |
| `phases` | List of phase numbers generated by planning |
| `current_phase` | Next phase to execute |

### `app/.agents/agents.json`

Agent configurations. See the [Agents Guide](./agents.md) for details.

```json
{
  "agents": [
    {
      "name": "Frontend Developer",
      "email": "frontend@myapp.local",
      "skills": ["frontend"],
      "scripts": [],
      "model": "sonnet"
    }
  ]
}
```

### `app/.agents/Skills.md`

A Markdown file that documents the skills and scripts available to agents. Organized with `## skill-name` sections and `### script-name.sh` subsections. See the [Agents Guide](./agents.md) for the full format.

### `app/.agents/CONTEXT.md`

Auto-generated during `orai plan`. Contains a snapshot of the project that agents read before starting any task:

- Tech stack (detected from config files)
- Directory tree
- Key documentation (first 80 lines of README, CLAUDE.md, etc.)
- Config file contents
- Entry point files the agent should read first

This file is regenerated each time you run `orai plan`. You do not need to edit it by hand.

### `app/.agents/tasks/phase1.json`

One file per phase, created during `orai plan`. Contains the phase metadata and all its tasks:

```json
{
  "phase_number": 1,
  "title": "Foundation Setup",
  "description": "Set up project structure, database, and authentication",
  "validation": "All config files exist, database connects, auth flow works",
  "tasks": [
    {
      "id": "1.1",
      "description": "Set up Prisma schema with User model",
      "prompt": "Read src/app/layout.tsx first to understand the project structure...",
      "priority": 1,
      "target_model": "sonnet",
      "target_agent": null,
      "status": "pending",
      "depends_on": [],
      "allowed_tools": ["Bash", "Edit", "Read", "Write", "Glob", "Grep"],
      "max_budget_usd": 1.0,
      "started_at": null,
      "completed_at": null,
      "error": null,
      "output_summary": null
    }
  ]
}
```

**Task fields:**

| Field | Description |
|-------|-------------|
| `id` | Unique ID in format `{phase}.{task}` (e.g., `1.3`) |
| `description` | Short summary of what the task does |
| `prompt` | Full instruction sent to the AI |
| `priority` | 1-10, where 1 is highest priority (executed first) |
| `target_model` | Which model to use (`haiku`, `sonnet`, or `opus`) |
| `target_agent` | Agent name to use (mutually exclusive with `target_model`) |
| `status` | Current state: `pending`, `running`, `done`, `failed`, `paused`, `skipped` |
| `depends_on` | List of task IDs that must complete before this task starts |
| `allowed_tools` | Which tools the AI can use during this task |
| `max_budget_usd` | Cost limit per task |
| `started_at` | Timestamp when execution started |
| `completed_at` | Timestamp when execution finished |
| `error` | Error message if the task failed |
| `output_summary` | First 500 characters of output if the task succeeded |

### `app/.agents/scripts/`

A directory for custom bash scripts that agents can run. You create these yourself and document them in Skills.md.

Example: `app/.agents/scripts/run-tests.sh`

```bash
#!/bin/bash
npm test -- --watchAll=false
```

### `app/.agents/pause`

A sentinel file. When this file exists, the runner stops after the current task. Created by `orai pause`, removed by `orai run --resume`.

## Template-specific files

### Next.js template

```
app/
  package.json
  tsconfig.json
  tailwind.config.ts
  src/
    app/
      layout.tsx
      page.tsx
      globals.css
      api/
    components/
  public/
```

### Python template

```
app/
  pyproject.toml
  src/
    {project_name}/
      __init__.py
      main.py
  tests/
    __init__.py
    test_main.py
```
